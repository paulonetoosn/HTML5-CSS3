Desafio 09

  • Centralizar vídeos

  • Incorporando videos extenos no html5

  • Assim ficou o código para incorporar os videos externos e centralizar

                <<div style="display: flex; justify-content: center;">>
                    <<div style="margin-right: 10px;">>
                        <<iframe width="560" height="315" src="https://www.youtube.com/embed/F7KzJ7e6EAc?si=SaOBp8t4o2p9c3f5" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerpolicy="strict-origin-when-cross-origin" allowfullscreen><</iframe>>
                    <</div>>
            
                    <<div>>
                        <<iframe width="560" height="315" src="https://www.youtube.com/embed/8mei6uVttho?si=NYvhtTN7HXIgQXcy" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerpolicy="strict-origin-when-cross-origin" allowfullscreen><</iframe>>
                    <</div>>

Explicação sobre o código

  • Divisão em blocos: O código está dividido em quatro blocos, cada um contendo dois vídeos do YouTube.
  • Flexbox: Cada bloco é envolto por uma <<div>> com estilo display: flex; justify-content: center;. Isso utiliza o modelo de layout flexível do CSS para centralizar os vídeos horizontalmente na página.
  • Estilo do primeiro vídeo: Cada vídeo é colocado dentro de uma <<div>> adicional. O primeiro vídeo em cada bloco tem um estilo margin-right: 10px;, que adiciona uma margem à direita para criar um espaço entre os vídeos.
  • Incorporação de vídeos do YouTube: Cada vídeo é incorporado usando a tag <<iframe>>. O atributo src contém o link incorporado do vídeo do YouTube. Outros atributos, como width, height, title, allow, referrerpolicy, frameborder e allowfullscreen, controlam vários aspectos da incorporação do vídeo, como dimensões, título, permissões e comportamento.
  • Permissões de reprodução: O atributo allow especifica quais funcionalidades são permitidas no vídeo incorporado, como autoplay, picture-in-picture e fullscreen. Isso determina como o vídeo pode ser reproduzido e interagido pelo usuário.
  • Política de referência: O atributo referrerpolicy especifica como o navegador deve enviar o cabeçalho HTTP Referer ao carregar o recurso do vídeo. Neste caso, está definido como strict-origin-when-cross-origin, o que significa que o referer será enviado apenas para solicitações no mesmo site, mas não para solicitações entre diferentes sites.
  • No geral, este código incorpora múltiplos vídeos do YouTube em uma página HTML e os organiza lado a lado usando o modelo de layout flexível. Cada vídeo pode ser reproduzido e interagido pelos usuários de acordo com as permissões definidas.
